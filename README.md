# KidGuard — 幼儿园冲突风险管理分析系统

中山大学大创项目 | 电子与信息工程学院

## 项目简介

基于计算机视觉，通过骨骼关键点空间几何关系构建规则库，
实现幼儿园场景下冲突行为的轻量化识别与风险管理分析。

### 核心功能
- **骨骼关键点提取**：使用YOLOv8n-pose模型实时提取人体骨骼关键点
- **冲突行为识别**：基于骨骼关键点的空间几何关系和动作特征进行冲突检测
- **风险管理分析**：评估冲突风险等级并生成详细报告
- **事件记录**：自动记录冲突事件的发生时间、持续时间和涉及人员
- **数据驱动**：使用熵权法进行特征权重计算，实现科学赋权

## 环境配置

```bash
conda create -n fightguard python=3.8
conda activate fightguard
pip install -r requirements.txt
```

## 运行方式

### 阶段一：数据驱动的特征提取与赋权
```bash
# 提取特征数据用于熵权法计算
python scripts/extract_features_eda.py

# 使用熵权法计算特征权重
python scripts/calculate_entropy_weights.py
```

### 阶段二：骨骼数据规则验证
```bash
python scripts/run_skeleton_interaction.py
```

### 阶段三：视频端到端检测
```bash
python scripts/run_video_end2end.py
```

## 技术实现

### 1. 骨骼关键点检测
- 使用YOLOv8n-pose模型进行实时骨骼关键点提取
- 支持COCO-17标准的17个关键点
- 适配NTU RGBD数据集和实时视频输入

### 2. 冲突检测算法
- **状态机**：使用有限状态机（FSM）进行冲突状态管理，解决瞬间噪点导致的误报问题
- **动作特征**：提取腕部加速度、相对接近速度、肘部角加速度、躯干倾角变化等物理特征
- **科学赋权**：使用熵权法计算特征权重，实现数据驱动的客观赋权
- **对称检测**：同时以A攻B和B攻A两种视角进行检测，提高召回率

### 3. 数据处理流程
1. **数据输入**：读取骨骼数据或视频文件
2. **预处理**：归一化关键点坐标，构建骨骼轨迹
3. **人员配对**：计算人员间距离，选择最可能交互的配对
4. **特征提取**：计算动作特征和攻击距离
5. **规则判定**：基于状态机和评分系统进行冲突判定
6. **结果输出**：生成事件记录和评估报告

## 配置参数

主要配置参数位于 `configs/default.yaml`：

- **规则阈值**：接近阈值、冲突持续帧数、手腕侵入阈值等
- **状态机参数**：各状态转换的帧数要求
- **输出设置**：事件保存、指标计算、可视化等
- **数据集定义**：冲突动作和正常动作的类别

## 应用场景

该系统可应用于：

- 任何需要监测人员冲突的场景

## 性能特点

- **轻量化**：使用YOLOv8n-pose轻量级模型，适合CPU运行
- **实时性**：优化的算法设计，确保实时检测
- **准确性**：基于物理特征和状态机的综合判定
- **可解释性**：详细的触发规则记录，便于理解检测结果
- **可扩展性**：模块化设计，易于添加新的检测规则和特征

## 关于在 带有Intel集成显卡的 华为轻薄本上，发挥OpenVINO工具优越性、加速YOLOv8-Pose推理   的经验略谈（实测时间减半）

## 亲爱的朋友们：

### 前言：为什么你该关心这个？

你现在跑 `python scripts/eval_video_dataset.py` 是不是觉得有点慢？  
我实测：20 个测试视频跑了 **2 分 55 秒**。调一次参数就要等将近 3 分钟，一天迭代十次就是半小时干等——太耽误事了。

**根本原因**：我用的是一台华为轻薄本（Intel Core Ultra + 集成显卡），PyTorch 默认用的是 CPU 推理，没有调用显卡和 NPU 的算力，等于让一个博士生去干抄写员的活，硬件实力没发挥出来。

**直接原因**：我想让每次跑测试的时间缩短一半，这样一天就能多跑几轮实验，早点出结果。

**好消息**：我找到了一个**不改代码逻辑、只换推理后端**的方法，实测时间缩短了 **50%**（2分55秒 → 1分27秒）。  
而且，如果你的笔记本也是 **Intel 芯片 + 集成显卡**（近三年买的轻薄本大概率都是），这个方法同样适用。  
如果你用的是 NVIDIA 独显且已经配好了 CUDA，那你已经很快了，不用看下去。

---

### 这是什么？

**OpenVINO** —— Intel 官方出的推理加速工具。

用人话解释：  
- PyTorch 推理像是一个翻译官，把模型的指令翻译成 CPU 能听懂的“普通话”。翻译过程本身挺慢，而且 CPU 也不是专门干这个的。  
- OpenVINO 是 Intel 自己家的翻译官，它提前把模型“背熟”（编译成硬件直接能跑的指令），还能把活分给集显甚至 NPU 一起干。所以跑得快，还省电。

**重点**：加速后的输出结果（关键点坐标、置信度）**和原来一模一样**，只是计算过程变快了。

### 不用 vs 用了（真实数据，我的电脑）

| 场景 | 不用 OpenVINO | 用 OpenVINO |
|------|--------------|-------------|
| 20 个视频总耗时 | 2 分 55 秒 | **1 分 27 秒** |
| 平均每帧耗时 | ~50 毫秒 | **~25 毫秒** |
| 代码改动量 | — | **只改 1 行** |
| 输出结果 | 关键点坐标 | **完全一样** |

加速比：**2 倍**。跑一次测试省下一半时间。

### 怎么操作？（三步，5 分钟搞定）

#### 第 0 步：确认你的电脑能用吗？
- **能用**：笔记本是 Intel 芯片（Core i5/i7/i9、Ultra 系列），带集成显卡（近三年买的轻薄本基本都是）。
- **不用换**：你有 NVIDIA 独显（RTX 30/40 系列）且已经在用 CUDA 推理。
- **不确定**：直接试，不行再回退，不影响原有代码。

#### 第 1 步：安装 OpenVINO（只需一次）
```bash
pip install openvino
```

#### 第 2 步：导出模型（只需一次）
新建一个 Python 文件，或直接在终端逐行输入：
```python
from ultralytics import YOLO
model = YOLO("yolov8n-pose.pt")
model.export(format="openvino", half=True)
```
运行后，当前目录会出现 `yolov8n-pose_openvino_model/` 文件夹。

#### 第 3 步：改你的脚本（只改 1 行）
把原来加载模型的那行：
```python
model = YOLO("yolov8n-pose.pt")
```
换成：
```python
model = YOLO("yolov8n-pose_openvino_model/")
```
其他代码**一字不改**，直接运行 `python scripts/eval_video_dataset.py`。

### 可能遇到的坑（极简版）

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 第一次跑卡了 10~20 秒 | OpenVINO 在编译模型，正常 | 等它跑完，下次就秒开了 |
| 报错 `Device not found` | 显卡驱动太老 | 去 Intel 官网更新 Intel Arc 驱动 |
| 报错 `No module named openvino` | 没安装 | 运行 `pip install openvino` |
| 速度没变快 | 没插电源 / CPU 降频 | 插上电源，电源模式选“最佳性能” |

### 后语：到底要不要用？

**强烈建议**：  
- 如果你也是 **Intel 轻薄本用户**，花 5 分钟配置一下，以后每次跑测试都省一半时间。省下来的时间可以多跑几轮实验、多调几次参数，或者早点下班。

**不用折腾**：  
- 如果你用的是 NVIDIA 独显 + CUDA，你已经很快了，保持现状即可。
- 如果你不确定自己的电脑是什么配置，直接试一下——失败了也不影响原有代码，删掉 `yolov8n-pose_openvino_model/` 文件夹就能回退。

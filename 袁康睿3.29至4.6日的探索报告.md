# **KidGuard 幼儿园冲突检测系统**  
## **阶段性技术进展与问题分析报告（定稿）**  
**截至 2026‑04‑06**

---

## 一、项目目标与技术路线

本项目面向幼儿园监控视频，构建基于人体骨骼序列的**冲突风险自动检测系统**，验收指标如下：

| 指标 | 目标值 |
|------|--------|
| Accuracy | 0.8–0.9 |
| Precision | 0.6–0.7 |
| Recall | 0.8–0.85 |
| FPR | < 0.15 |
| FNR | < 0.2 |

**技术路线（双链路并行）：**

- **NTU RGB+D 3D 骨骼链路**：验证物理建模与规则体系的正确性及理论上限  
- **真实 2D 视频链路（YOLOv8‑Pose + ByteTrack）**：验证系统在真实场景中的可落地性  

**核心方法框架：**  
> 物理建模（规则 + 状态机） → 特征提取 → 决策输出

---

## 二、已完成工作与当前性能

### 2.1 规则系统实现

- 完整实现队长提出的物理建模体系：肩宽归一化、双向交互、同步作用‑响应因果链、四阶段状态机、置信度抑制与时间窗确认  
- 关键修复：NTU 读取与时序对齐 Bug、归一化错误、单人样本污染  
- 新增 NTU 专用评测脚本，补齐 FNR 指标  

### 2.2 NTU 3D 骨骼链路评测结果（90 个有效双人样本）

| 指标 | 当前值 |
|------|--------|
| Accuracy | 0.556 |
| Precision | 0.302 |
| Recall | 0.842 |
| FPR | 0.521 |
| FNR | 0.158 |

**观察结论：**  
- 冲突召回能力强（Recall > 0.84）  
- 误报率高，正常互动（拥抱、拍背、握手）大量被判为冲突  
- 规则对“强烈互动”敏感，但对“互动语义”缺乏区分能力  

### 2.3 真实 2D 视频链路评测结果（FiveFight）

- Recall 可达 0.8–1.0  
- FPR 稳定在 0.4–0.5，难以降低  
- Accuracy 通常 ≤ 0.6  

**主要问题：** 无 Z 轴深度、关键点抖动/遮挡、追踪 ID 不稳定、运动学特征重叠严重  

---

## 三、关键问题与根因分析

通过双链路验证，确认以下事实：

1. **规则实现正确**：状态机可触发，物理量计算逻辑成立，异常结果可追溯到数据或语义层面  
2. **规则系统已达物理层面能力上限**：对“强烈身体互动”敏感，但对“是否冲突”缺乏判别力  
3. **运动学特征天然不可分**：拥抱、推搡、拍背等在速度、加速度、位移上高度重叠，仅靠阈值或逻辑无法稳定区分  
4. **数据与任务不匹配**：NTU 是动作识别数据集，未针对“冲突语义”设计；真实场景语义依赖上下文  

> **结论：当前瓶颈源于规则方法的理论局限性，而非工程或实现缺陷。**

## 五、方法局限性说明

- 规则判定对语义无感（无法区分善意互动与冲突）  
- 阈值不可跨域复用（3D / 2D / 不同视角下物理量分布差异大）  
- 对真实视频噪声敏感（遮挡、追踪错误破坏规则连续性）

## 六、下一步工作规划

### 6.1 核心策略：从“规则判决”转向“规则特征 + 学习模型”

采用**多路线并行、小步快跑**策略，不再继续押注单一调参。

### 6.2 推荐路线（按优先级排序）

| 优先级 | 路线 | 核心思想 | 预期收益 |
|--------|------|----------|----------|
| ★★★★★ | 规则特征化 + SVM / XGBoost | 规则提取特征，轻量分类器学习决策边界 | Accuracy ≥ 0.8，FPR 显著下降 |
| ★★★★☆ | 规则候选筛选 + 二级精判 | 第一级高召回，第二级 ML 精判 | 工程结构清晰，适合真实部署 |
| ★★★☆☆ | 规则特征 + 简单时序模型 | 利用时间结构进一步区分 | SVM 卡住时的备选 |
| ★★★☆☆ | 纯规则 + 数据驱动阈值学习 | 自动搜索最优阈值 | 仅作对照实验 |
| ★★☆☆☆ | ST-GCN / GNN | 端到端图神经网络 | 数据不足、风险高，适合未来工作 |
| ★☆☆☆☆ | 继续打磨纯规则 | 反复调参 | 已被证明不可行 |

### 6.3 主线方案（推荐严格执行）

#### ✅ **路线①：规则特征化 + 传统机器学习分类器**

**实施步骤：**

1. **重构规则输出接口**：对每个交互片段导出结构化特征（示例：`max_r_a, max_r_v, mean_r_v, max_r_phi, max_r_p, duration, response_frame_ratio, mean_gamma`，可扩展峰值次数、不对称性等）  
2. **划分数据集**：开发集（调参）、验证集（比较）、盲测集（最终检验）  
3. **训练分类器**：优先 SVM（RBF核）、Random Forest、XGBoost，使用 5‑fold 交叉验证  
4. **评估与迭代**：若 Accuracy ≥ 0.8 且 FPR < 0.2 则固化；若仍卡在 FPR，则增加时序特征或尝试路线③  

**选择理由：**  
- 已有高质量物理特征  
- 已证明人工阈值不可行  
- 小样本 + 高相关特征场景下 SVM/XGBoost 表现强  
- 工程成本低，与现有代码兼容  

### 6.4 辅助工程建议

- **误报类型学整理**：将误报分为拥抱、拍背、追逐、关键点漂移、ID 混乱等，指导特征设计  
- **输入质量判定**：对 2D 片段增加平均置信度、ID 连续性、肩宽稳定性等指标，低质量片段降低决策权重  
- **跨域对照实验**：固定特征提取方式，对比 NTU 与 FiveFight 上的特征分布，解释“规则难以直接落地”的根源  
- **轻量深度学习备选**：MLP / LSTM 仅作对照，不作为主线
